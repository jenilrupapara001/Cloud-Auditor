# Frequently Asked Questions (FAQ)

### 1. Is my AWS data safe?
Yes. Cloud Auditor runs **100% locally** on your machine. It never sends your AWS credentials, resource metadata, or scan results to any external servers. Your security posture remains within your perimeter.

### 2. Can I see the source code?
No. Cloud Auditor is proprietary software. We keep the source code private to ensure the integrity of our hardening techniques and to protect our intellectual property.

### 3. How does RSA Signing work?
Every report generated by Cloud Auditor is cryptographically signed using an internal private key. You can use our public verification tool (coming soon) to ensure that the report has not been tampered with since generation.

### 4. What is Hardware Binding?
Enterprise licenses are tied to the unique hardware ID of the machine where the CLI is installed. This prevents unauthorized redistribution of licensed binaries.

### 5. How do I get support?
Please open an [Issue](https://github.com/USER/cloud-auditor/issues) on this repository for bug reports or feature requests. For enterprise support, please refer to your commercial agreement.

### 6. Do you support multi-cloud?
Currently, we offer exhaustive support for **AWS**. Azure and Google Cloud support is on our roadmap for later this year.
